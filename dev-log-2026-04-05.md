## Summary

Created a real-time weather monitoring web application that displays current weather conditions for 21 major cities worldwide. Users can view weather data through a responsive web interface with search and sorting capabilities.

## Features Implemented

### Core Features
- Real-time weather data for 21 major cities globally
- Beautiful dark-themed responsive UI with glassmorphism effects
- Temperature unit toggle (Celsius/Fahrenheit)
- City search functionality
- Auto-refresh every 5 minutes

### Sorting Feature (Added)
- Sort by city name (A-Z, Z-A)
- Sort by country name (A-Z, Z-A)
- Sort by temperature (Low to High, High to Low)

### Weather Details Displayed
- Current temperature
- Feels like temperature
- Humidity percentage
- Wind speed (km/h)
- Atmospheric pressure (hPa)
- Weather description with icon

## Development Timeline

### 1. Initial Setup
- Created Flask backend with OpenWeatherMap API integration
- Built responsive frontend with dark theme
- Implemented basic weather card display

### 2. Issue: CSS Not Loading
- **Problem:** Page displayed without styling (plain HTML)
- **Cause:** User opened HTML file directly instead of through Flask server
- **Solution:** Instructed to access via `http://localhost:PORT`

### 3. Issue: API Key Required
- **Problem:** OpenWeatherMap requires API key registration
- **Solution:** Switched to Open-Meteo API (completely free, no registration needed)

### 4. Issue: Port 5000 Blocked (HTTP 403)
- **Problem:** macOS AirPlay Receiver uses port 5000
- **Solution:** Changed to port 3000 with host `127.0.0.1`

### 5. Issue: Python Syntax Error
- **Problem:** `dict | None` syntax not supported in Python < 3.10
- **Solution:** Changed to `Optional[dict]` using typing module

### 6. Performance Optimization
- **Problem:** Sequential API calls for 21 cities was slow
- **Solution:** Implemented parallel requests using `ThreadPoolExecutor`

### 7. Feature Addition: Sorting
- Added dropdown menu for sorting options
- Implemented sort by city, country, and temperature

## Lessons Learned

1. **macOS Port Conflicts:** Port 5000 is used by AirPlay Receiver on macOS Monterey+
2. **Python Version Compatibility:** Use `Optional[T]` instead of `T | None` for Python < 3.10
3. **Free Weather APIs:** Open-Meteo is an excellent free alternative to OpenWeatherMap
4. **Performance:** Use `ThreadPoolExecutor` for parallel HTTP requests to improve response time
